# CSRF Attack Demo

Демонстрация уязвимости Cross-Site Request Forgery (CSRF) на примере банковского веб-приложения.

## Что такое CSRF?

CSRF — это атака, которая заставляет пользователя выполнить нежелательные действия в веб-приложении, в котором он аутентифицирован. Браузер автоматически подставляет cookies сессии при межсайтовых запросах.

## Как запустить

1. Установите зависимости:
```bash
npm install
```

2. Запустите сервер:
```bash
npm start
```

3. Откройте браузер и перейдите на http://localhost:3000

## Демонстрация атаки

1. Войдите в банковское приложение (любой логин/пароль)
2. Увидите баланс 10000 рублей
3. Откройте новую вкладку и перейдите на http://localhost:3000/hacker-attack.html
4. Через 2 секунды автоматически выполнится CSRF-атака
5. Вернитесь к банковскому приложению — баланс уменьшится на 1000 рублей

## Файлы

- `server.js` — Express сервер с уязвимым банковским приложением
- `public/vulnerable-bank.html` — уязвимое банковское приложение
- `public/hacker-attack.html` — страница злоумышленника

## Уязвимости в демо

- Отсутствие CSRF-токенов
- Cookies без атрибута SameSite
- Нет проверки Origin/Referer заголовков

## Меры защиты

- CSRF-токены
- SameSite cookies
- Проверка Origin/Referer
- Double-submit cookie pattern
